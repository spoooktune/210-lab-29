To unit test my program, I would alter my code in main. Specifically, 
I would alter how many items are created on Day 1, as well as the
chance for each event to occur. The variable r that controls the randomness
of each event would stay at 100, and the condition for each to occur would also
be at 100. I would then add 3 objects to the map and run the program. The 
program already outputs a message whenever a change occurs, as well as a 
summary after each day and after the simulation is over. Therefore, it is
very easy to see whether the program has worked properly.

        int r = 0;

        // for loop that iterates through each item in map
        for (auto &pair : store){
            // set r to random int (1-100)
            // r = rand() % 100 + 1;
            // if r meets condition for price hike

                // call price hike func
                cout << "Price for " << pair.first;
                price_hike(pair.second);
                event = true;
                break;
            }
            // set r to random int (1-100)
            // r = rand() % 100 + 1;
            // if r meets condition for sale
            if (r == 100)
                // call sale func
                cout << pair.first;
                sale(pair.second);
                event = true;
                break;
            }
            // set r to random int (1-100)
            // r = rand() % 100 + 1;
            // if r meets condition for rating decrease
            if (r == 100){
                // call rating decrease func
                cout << pair.first;
                rating_decrease(pair.second);
